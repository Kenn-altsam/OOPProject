Research-Oriented University System — Documentation

This documentation describes the main 5-10+ classes, interfaces, patterns and console demo.

Architecture

  • project.models contains the model classes that correspond to the UML class/object diagram.
  • project.storage contains DataRepository, AppData, and FileStorage for serialization.
  • project.ui contains console menus.
  • project.patterns contains Factory, Strategy and Decorator implementations.

Core Classes

CreateUser

Abstract superclass for all users. Stores id, username, password, name, email, role. Implements authentication, Comparable, equals, hashCode, and toString.

CreateEmployee

Base class for employees. Adds department, inbox, complaints, sendMessage(), and sendComplaint().

CreateTeacher

Teacher role. Can view courses, view students, put marks, receive ratings, and act as Researcher. Professors automatically become researchers with h-index at least 3.

RegisterStudent

Student role. Can register for courses, view marks, get transcript, rate teachers, and receive course notifications. Checks 21-credit limit and fail-count rule. Fourth-year supervisor requires h-index >= 3.

AppointManager

Manager role. Approves registration, assigns teachers to courses, creates statistical course reports, and sorts students by GPA.

AdministerSystem

Admin role. Adds, updates, removes users, and views access logs.

EstablishCourse

Course model. Supports multiple instructors, enrolled students, lessons, and Observer pattern methods attach(), detach(), notifyObservers().

GenerateGrade

Mark model. Stores attestation 1, attestation 2, and final exam. Calculates total, letter grade and GPA point.

PublishResearchPaper

Research paper object. Stores title, authors, journal, DOI, publication date, citations, pages and keywords. Implements Comparable.

InitiateProject

Research project. Stores topic, participants, papers and start date. Throws NotResearcherException when a non-researcher tries to join.
